# 現状まとめ 2 - ollama API + 報告書生成の実装

前回の「現状をまとめた.md」から、以下の機能追加・改善を実施しました。本ドキュメントは、2026年5月14日時点での実装状態をまとめたものです。

## 最新の目的

Git の未コミット差分を取得し、ollama API（ローカル LLM）で可読性の高い変更報告書を生成し、**プロジェクトごと・タイムスタンプ付きで output フォルダに自動保存する**ようになりました。

変更内容の記録が残りやすくなり、プロジェクト単位での履歴管理や、チーム間での共有が容易になっています。

## 新機能

### 1. 報告書生成機能（`src/generateReport.ts`）

```typescript
export async function generateDiffReport(options: ReportOptions = {}): Promise<string>
```

- `getDiff()` で未コミット差分を取得
- ollama API に投げて詳細なマークダウン形式の報告書を生成
- プロジェクト名 + タイムスタンプを付けてファイル保存
- 出力ファイル例: `output/yoshida_zemi_project_20260514_180702.md`

### 2. 改善されたプロンプト（`src/prompt.ts`）

前: 「以下の内容を簡潔に要約してください」（シンプルな要約）

後: 「git diff を分析して、マークダウン形式の詳細な変更報告書を作成してください」（構造化された報告書）

出力フォーマット指示:
- 変更ファイルごとに説明
- 何が変わったか（概要）
- 追加された主な機能や改善
- 削除された部分がある場合その理由
- 影響を受ける可能性のあるコンポーネント
- 全体的な変更の意図と目的
- 注意点やリスク（ある場合）

### 3. プロジェクト情報の自動抽出

- 実行時のカレントディレクトリから自動でプロジェクト名を取得
- タイムスタンプを `YYYYMMDD_hhmmss` 形式で付加
- ISO 8601 形式の生成日時をレポート内に記載

### 4. テスト用 HTTP サーバー（`test_project/server.cjs`）

Web UI からの動作確認用に、以下のエンドポイントを提供：

- `GET /` → テスト HTML ページ
- `GET /raw` → 生の `git diff` を返す（text/plain）
- `GET /summary` → ファイルごとの +/- 行数集計（JSON）
- `GET /report` → レポート生成のステータス情報（現在は placeholder）

### 5. 改善されたビルド設定（`tsconfig.json`）

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ES2020",
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "esModuleInterop": true
  }
}
```

TypeScript を正しく dist/ にビルドして、Node.js で実行可能にしました。

## 動作の流れ（更新版）

```
1. main.ts 起動（プロジェクト名を引数で受け取る）
   ↓
2. generateDiffReport() が呼ばれる
   ↓
3. getDiff() → git diff を取得
   ↓
4. buildSummaryPrompt() → マークダウン形式の詳細プロンプトを生成
   ↓
5. generate() → ollama API に投げて報告書を生成
   ↓
6. buildReportTemplate() → プロジェクト情報とタイムスタンプを付加
   ↓
7. fs.writeFile() → output/{projectName}_{timestamp}.md に保存
```

## 実行方法

### 1. 初回セットアップ

```bash
# 依存関係をインストール（必要に応じて）
npm install

# TypeScript をコンパイル
npm run build
# または
node -e "require('child_process').execSync('npx tsc', { stdio: 'inherit' })"
```

### 2. 報告書を生成

```bash
# 基本的な使い方
node dist/main.js "プロジェクト名"

# 出力ディレクトリを指定する場合
node dist/main.js "プロジェクト名" "カスタム出力パス"
```

例:
```bash
node dist/main.js "yoshida_zemi_project"
# → output/yoshida_zemi_project_20260514_180702.md が生成される
```

### 3. テスト用サーバーの起動（オプション）

```bash
node test_project/server.cjs
# Server listening http://localhost:3000
```

ブラウザで http://localhost:3000 にアクセス

## 依存環境

- **Node.js 24.x** 以上
- **TypeScript 5.8.x** 以上
- **Ollama** が `http://localhost:11434` で動作していることが必須
- 使用モデル: `qwen2.5-coder:1.5b`（src/ollama.ts で指定）

Ollama がない場合:
```bash
# https://ollama.ai から Ollama をダウンロード・インストール
ollama pull qwen2.5-coder:1.5b
ollama serve
```
