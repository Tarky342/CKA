# 現状まとめ

このリポジトリは、Git の差分を取得して LLM に渡し、要約を Markdown ファイルへ保存する小さな TypeScript CLI です。現時点では「コード変更を知識化するエージェント」の最小構成として動く段階で、構想文書にある大きな設計のうち、差分取得と要約生成の入口だけが実装されています。

## 目的

現在の目的は、作業中のリポジトリにある変更内容を自動で読み取り、短く分かりやすい要約として出力することです。人が diff を直接読む負担を減らし、変更内容の把握を速くするための補助ツールとして作られています。

## 動作の流れ

1. `main.ts` が起動する。
2. `git diff` を実行して、未コミットの変更を文字列として取得する。
3. `prompt.ts` が、その差分を要約させる日本語プロンプトを作る。
4. `ollama.ts` がローカルの Ollama API に対して要約を依頼する。
5. 返ってきた文章を、指定された出力先に書き込む。

## 主要ファイル

- `package.json`
  - 実行スクリプトを定義している。
  - 開発時は `tsx src/main.ts` で起動する。
  - TypeScript と Ollama 呼び出し用の最小構成になっている。

- `src/main.ts`
  - 全体のエントリポイント。
  - 出力先パスをコマンドライン引数から受け取る。
  - diff の取得、プロンプト生成、要約生成、ファイル保存を順番に実行する。

- `src/git/getDiff.ts`
  - `git diff` をシンプルに実行して差分を取得する。
  - 現状は未コミット差分だけを対象にしている。

- `src/prompt.ts`
  - 取得した内容を「簡潔に要約してください」という形のプロンプトに変換する。
  - ここで要約の指示文を作っているため、出力品質に直接影響する。

- `src/ollama.ts`
  - `http://localhost:11434/api/generate` に POST して要約を生成する。
  - 使用モデルは `qwen2.5-coder:1.5b`。
  - ローカル Ollama が動いていないと失敗する。

- `src/typece.ts`
  - 現時点では空のファイル。
  - 型定義の置き場として用意された可能性があるが、まだ使われていない。

- `input/sample.txt`
  - サンプル入力テキスト。
  - ただし、現行コードではこのファイルを読み込んでいない。

- `output/summary.md`
  - サンプル出力の保存先として使われている。
  - 実行時のデフォルト出力先にもなっている。

## プロジェクトの位置づけ

上位の構想文書では、このプロジェクトは「Git の変更履歴を解析して知識化するエージェント」の初期版として説明されています。将来的には、単純な diff 要約だけでなく、構造化イベントの保存、意図の推論、Symbol レベルの解析、Markdown への知識化などへ広がる設計です。

今のコードはその中でもかなり小さく、まずは以下を確認するための試作になっています。

- Git の差分を正しく取れるか
- Ollama に投げて文章を返せるか
- その結果をファイルへ保存できるか

## 現状の特徴

- 入力は基本的に `git diff` だけで、コミット履歴や AST 解析はまだ使っていない。
- 出力は Markdown 形式のテキストで、人がすぐ読めることを優先している。
- 構成が非常に小さいため、理解しやすく、実験もしやすい。
- ただし、要約の品質は LLM の応答に強く依存する。
- ローカル API 前提なので、Ollama が停止していると動かない。

## 注意点

- `tsconfig.json` は空で、まだ厳密な TypeScript 設定は入っていない。
- `src/typece.ts` も空のため、型設計は未整備。
- `input/sample.txt` はあるが、現状の処理フローでは使われていない。
- `git diff` をそのまま渡すため、差分が大きいと要約の精度が落ちる可能性がある。

## 一言でまとめると

このリポジトリは、「Git の未コミット差分をローカル LLM で要約して Markdown に保存する、最小構成の TypeScript CLI」です。
